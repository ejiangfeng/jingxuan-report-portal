## 🔧 API接口使用

### 健康检查
```bash
curl http://localhost:4000/api/v1/health
```

### 获取筛选选项
```bash
curl http://localhost:4000/api/v1/orders/filter-options
```

### 订单查询
```bash
curl -X POST http://localhost:4000/api/v1/orders/query \
  -H "Content-Type: application/json" \
  -d '{
    "page": 1,
    "pageSize": 20,
    "filters": {
      "dateRange": {
        "start": "2026-01-01",
        "end": "2026-02-16"
      },
      "storeId": "1101"
    }
  }'
```

### 订单统计
```bash
curl -X POST http://localhost:4000/api/v1/orders/stats \
  -H "Content-Type: application/json" \
  -d '{
    "filters": {
      "dateRange": {
        "start": "2026-01-01",
        "end": "2026-02-16"
      }
    }
  }'
```

## 📋 使用技巧

### 💡 查询优化技巧
1. **日期范围** - 尽量缩小查询时间范围
2. **分页使用** - 大数据量时使用分页查询
3. **组合筛选** - 多个条件组合提高查询精度
4. **快捷查询** - 使用预定义的查询模板

### 🎯 导出最佳实践
1. **导出前筛选** - 精确筛选需要导出的数据
2. **分批次导出** - 大数据量时分批次导出
3. **格式选择** - Excel适合数据分析，CSV适合系统导入

### 🔄 性能提示
- 首次查询可能需要稍长时间
- 后续查询有缓存优化
- 数据量超过10万条时建议分页查询

## 🛠️ 常见问题解决

### ❓ Q1: 无法访问前端页面
```
✅ 解决方案:
1. 确认系统已启动: ./start-system.sh
2. 检查端口状态: lsof -ti:3001
3. 手动访问: http://localhost:3001/simple-frontend.html
```

### ❓ Q2: API接口返回错误
```
✅ 解决方案:
1. 检查后端服务: curl http://localhost:4000/api/v1/health
2. 查看后端日志: tail -f backend/backend.log
3. 重启服务: ./stop-system.sh && ./start-system.sh
```

### ❓ Q3: 查询数据为空
```
✅ 解决方案:
1. 确认筛选条件设置正确
2. 扩大时间范围
3. 清除筛选条件重新查询
```

### ❓ Q4: 前端TypeScript错误
```
📋 状态说明:
当前使用简化的HTML前端绕过TypeScript编译问题
完整React应用由于类型错误暂时无法编译

🔧 后续解决方案:
1. 修复前端的TypeScript配置
2. 安装缺失的类型定义 (@types/node)
3. 清理未使用的变量声明
```
